## 方式推荐

v2ray 翻墙方式，支持ss和vmess两种方式。

ss即shadowsocks，应该都很熟悉，这里不多做赘述。vmess可查看 [v2ray](https://www.v2ray.com/)

vmess宣称可以在ss失效的时候可以用，但以我个人的使用经验来看，ss失效的时候v2ray基本上也不可用。但这里还是推荐v2ray，因为他可以同时支持这两种方式，而且各平台的客户端都很齐全，且都有使用说明文档。

各平台客户端推荐如下：

Windows： [v2rayN](https://github.com/2dust/v2rayN)

Mac： [v2rayU](https://github.com/yanue/v2rayu)

Android: [Shadowsocks](https://play.google.com/store/apps/details?id=com.github.shadowsocks&hl=zh-CN)

iOS: [Shadowrocket](https://apps.apple.com/us/app/shadowrocket/id932747118)

## 一些有用的操作

### Git翻墙

单独设置git翻墙很多网上的设置可以找到很多，这里先贴出来，并不推荐这种方式，因为

1. 只对git单独设置代理，很多使用情况下我们是希望curl，homebrew其实也去用代理的。

2. git全局设置，当使用国内gitee等网站的时候也使用代理就没必要了。

设置代理：

```shell
git config --global http.proxy 'socks5://127.0.0.1:10808' 
git config --global https.proxy 'socks5://127.0.0.1:10808'
```

 

查看代理：

```shell
git config --global --get http.proxy
git config --global --get https.proxy
```

 

取消代理：

```shell
git config --global --unset http.proxy
git config --global --unset https.proxy
```

这里我觉得可以设置终端翻墙更合理点。

### 终端翻墙

其实从git设置代理的方式我很可以看出，**操作系统其实已经内置了代理的变量，http请求是否使用代理就是http.proxy和https.proxy是否指定变量**。Mac，Linux，和Windows变量名都一样。只要在终端里敲下这些命令，就可以在使用代理和不使用代理之间切换。另外，可以同时设置socks的代理，socks和http在网络协议不在同一层里，但它也是通过变量设置代理的，在Mac下具体可以按如下设置：

设置代理：

```shell
export https_proxy=http://127.0.0.1:1087; 
export http_proxy=http://127.0.0.1:1087; 
export ALL_PROXY=socks5://127.0.0.1:1080
```

查看代理：

```shell
echo $http_proxy
```

取消代理：

```shell
unset http_proxy; unset https_proxy; unset ALL_PROXY
```

*端口号根据你使用的软件上写的socks和http端口号。像我使用的软件sock端口是1080，http是1087*

另外，可以使用alias把这些命令封装一下

```shell
alias setproxy="export https_proxy=http://127.0.0.1:1087; export http_proxy=http://127.0.0.1:1087; export ALL_PROXY=socks5://127.0.0.1:1080"
alias unsetproxy="unset http_proxy; unset https_proxy; unset ALL_PROXY"
alias proxy="echo $http_proxy"
```

还有个简单的curl命令可以简单查看当前ip（可以查看是否翻墙已经可用了）

```shell
curl cip.cc
```

当然，alias下更方便

```shell
alias ip="curl cip.cc"
```

这样子，在终端翻墙就很简单了，打开终端，输入setproxy，然后就可以愉快的使用homebrew，curl，git翻墙了，让下载速度飞起来！如果不使用代理，输入unsetproxy就可以了，如果你不知道当前有没有使用代理，输入proxy可以查看当前的代理设置。

这里多说一句，android常用的gradle代理方式也是如此。

### 软件代理

常用的有浏览器（我用的是Edge，可以认为是Chrome），Android Studio ，Telegram的软件代理设置。

1. 对于浏览器来讲，其实它本身就有默认设置使用系统代理。所以你在使用v2rayN等软件的时候，只要在v2rayN软件指定了哪些链接使用代理，哪些链接直连（也就是所谓的pac设置），就可以了，浏览器不需要装额外的插件。pac规则一般GitHub上都有维护，直接找一个就行了。
2. 对于Android Studio来讲，它里面有很多东西比如gradle其实使用pac是会无效的，主要是很多东西你不知道改不改使用pac，（如果有知道的欢迎给我说明下）。所以在软件的proxy设置里面，要么选择不使用代理，要么选择使用代理就可以了
3. Telegram单独来讲其实有点多余，TG的代理逻辑其实就是电脑上大部分软件的代理逻辑，如果要使用代理就得单独在软件里设置，因为**软件默认不使用系统代理，除了浏览器**。
